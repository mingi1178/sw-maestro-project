## 프로젝트 의도

27조 — **AI 주식 리포트 생성 서비스**. **한국투자증권 API**를 통해 한국 주식 시세 데이터를 수집하여 시계열 분석을 수행하고, 정형화된 투자 리포트를 자동 생성하며, 후속 질문에는 대화형으로 응답하는 Agentic Workflow.

MVP 범위의 명확한 경계 (기획서 기준):
- 요청당 단일 종목, 과거/최근 시세 데이터만.
- 실시간 매매 실행 없음, 매수/매도 추천 없음.
- 복잡한 기술적 지표(RSI/MACD) 없음, 다중 증권사 API 연동 없음, 외부 리서치 리포트 요약 없음.

사용자가 위 제외 항목을 요청할 경우, 구현 전에 범위 경계를 먼저 알려야 합니다.

## 계획된 아키텍처

기획서는 다음 sub-agent들로 구성된 멀티 에이전트 시스템을 기술합니다 (오케스트레이션 프레임워크는 챕터 가이드에 명시된 LangGraph를 사용; docx에는 대안으로 LangChain/CrewAI도 언급되지만, 챕터 목차가 더 최신 결정으로 보임):

1. **Collector sub-agent** — 선택 종목의 시세/공시 데이터를 수집하고, **청킹(chunk)** 후 로컬 벡터 DB에 저장.
2. **Extractor sub-agent** — 주어진 질의에 대해 벡터 저장소에서 관련 정보를 추출.
3. **Reporter sub-agent** — 추출된 정보를 토대로 정형화된 투자 리포트(핵심 분석 요약, 투자 의견, 리스크) 작성.
4. **Chatbot sub-agent** — 자연어 라우팅 계층: 주식 관련 질의는 벡터 DB 검색 후 응답, 무관한 질의는 검색 없이 직접 응답.

계획된 스택 (챕터 4 빌드 순서):
- 4.1 저장소/환경 설정
- 4.2 Agentic Workflow Design
- 4.3 LLM Core 및 서비스 계층 (docx에는 Gemini API 언급; 구체 모델은 아직 미확정으로 간주)
- 4.4 Vector DB 및 내부 지식 베이스 검색
- 4.5 **LangGraph** 기반 멀티 에이전트 오케스트레이션
- 4.6 **FastAPI** 백엔드
- 4.7 **Streamlit** 프론트엔드
- 4.8 로컬 실행 및 **Docker** 패키징

데이터/분석 라이브러리: Pandas, Matplotlib/Plotly.
