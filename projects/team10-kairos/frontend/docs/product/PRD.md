# Kairos MVP PRD

## 1. 제품 개요

Kairos는 사용자가 자연어로 입력한 일정을 AI Agent가 분석해 캘린더 일정과 알림으로 등록해주는 모바일 생활 관리 앱이다.

제품의 핵심 가치는 **한 문장 입력으로 일정 관리 완료**이다. 사용자는 날짜, 시간, 제목, 장소, 알림 시간을 각각 직접 입력하지 않고 평소 말하듯 일정을 입력한다. Agent는 입력 문장에서 필요한 정보를 추출하고, 부족한 정보는 추가 질문으로 보완한 뒤, 사용자 확인을 받은 경우에만 일정과 알림을 등록한다.

## 2. 문제 정의

일정은 대화 중 자연스럽게 생기지만, 기존 캘린더 앱에 등록하려면 사용자가 여러 필드를 직접 채워야 한다.

현재 사용자는 보통 다음 과정을 직접 수행한다.

- 캘린더 앱 실행
- 날짜 선택
- 시작 시간과 종료 시간 입력
- 일정 제목 입력
- 장소 입력
- 알림 시간 선택
- 저장
- 필요한 경우 별도 알림 설정

이 과정은 단순하지만 반복되면 피로도가 높고, 바쁜 상황에서는 등록을 미루다가 일정을 잊는 문제가 생긴다. Kairos는 자연어 입력을 구조화된 일정 데이터로 변환하고, 캘린더 등록과 알림 예약까지 연결해 이 문제를 줄인다.

## 3. 목표 사용자

Kairos의 주요 사용자는 일정은 많지만 캘린더와 알림을 직접 관리하는 데 불편함을 느끼는 사람이다.

- 과제, 시험, 약속, 동아리 일정이 많은 대학생
- 회의, 미팅, 마감 일정이 많은 직장인
- 병원 예약, 운동, 개인 업무를 자주 잊는 일반 사용자
- 캘린더 앱은 필요하지만 직접 입력 과정을 귀찮아하는 사용자
- 자연어 입력으로 빠르게 일정 관리를 끝내고 싶은 사용자

## 4. MVP 목표

MVP는 모바일 앱에서 사용자가 자연어로 일정을 입력하고, Agent가 필요한 정보를 정리한 뒤, 사용자 승인 후 일정과 알림을 등록하는 전체 흐름을 검증하는 것을 목표로 한다.

### 성공 기준

- 사용자가 자연어 한 문장으로 일정 등록 흐름을 시작할 수 있다.
- Agent가 일정명, 날짜, 시간, 장소, 알림 시간을 추출할 수 있다.
- 필수 정보가 부족하면 Agent가 추가 질문을 할 수 있다.
- 등록 전 사용자에게 최종 내용을 요약해 확인을 받을 수 있다.
- 승인된 일정만 캘린더에 저장되고 알림이 예약된다.
- 등록 완료 후 사용자가 결과를 확인할 수 있다.

### KPI

| 지표                  | 목표          |
| --------------------- | ------------- |
| 일정 등록 성공률      | 90% 이상      |
| 일정 정보 추출 정확도 | 85% 이상      |
| 추가 질문 횟수        | 평균 1회 이하 |
| 등록 완료 시간        | 30초 이내     |

## 5. 핵심 사용자 시나리오

### 시나리오 1: 일정과 알림을 한 번에 등록

사용자가 "이번 주 토요일 오후 6시에 홍대에서 친구 만나. 1시간 전에 알려줘."라고 입력한다.

Agent는 날짜, 시간, 장소, 일정명, 알림 시간을 추출하고 등록 내용을 요약한다. 사용자가 승인하면 캘린더 일정과 알림을 등록하고 완료 메시지를 보여준다.

### 시나리오 2: 필수 정보가 부족한 일정 등록

사용자가 "내일 병원 가는 거 알림 맞춰줘."라고 입력한다.

Agent는 시간 정보가 없음을 감지하고 "몇 시 일정인가요?"처럼 추가 질문을 한다. 사용자가 "오후 3시"라고 답하면 Agent는 최종 등록 내용을 요약하고 사용자 승인을 요청한다.

### 시나리오 3: 마감 일정 등록

사용자가 "금요일 밤 12시까지 프로젝트 기획서 제출해야 해."라고 입력한다.

Agent는 입력을 마감 일정으로 인식하고 마감 시점을 기준으로 일정을 생성한다. 알림 시간이 명시되지 않았다면 기본 알림을 적용하고, 등록 전 사용자 확인을 받는다.

## 6. 주요 화면 요구사항

Figma selection은 홈/캘린더 화면 설계 참고 자료로 사용한다. PRD에서는 픽셀 단위 UI가 아니라 사용자가 각 화면에서 달성해야 하는 행동과 정보 요구사항을 정의한다.

### 홈 화면

- 사용자는 앱 진입 후 자연어 일정 입력을 바로 시작할 수 있어야 한다.
- 자연어 입력 영역은 홈 화면의 핵심 액션으로 보여야 한다.
- 사용자가 입력한 문장에 대해 Agent의 분석 상태, 추가 질문, 등록 전 요약을 같은 흐름 안에서 확인할 수 있어야 한다.
- 등록 전 요약에는 일정명, 날짜, 시간, 장소, 알림 시간이 표시되어야 한다.
- 사용자는 요약 내용을 승인하거나 취소할 수 있어야 한다.

### 캘린더 화면

- 사용자는 날짜 기준으로 등록된 일정을 확인할 수 있어야 한다.
- 선택한 날짜의 일정 목록을 확인할 수 있어야 한다.
- 새로 등록된 일정은 캘린더 화면에서 확인 가능해야 한다.
- 일정 목록에는 최소한 일정명, 시간, 알림 여부가 표시되어야 한다.
- 일정이 없는 날짜에는 빈 상태를 표시해야 한다.

### Agent 상호작용 상태

- 분석 중 상태: 사용자가 입력한 자연어를 Agent가 처리 중임을 보여준다.
- 추가 질문 상태: 필수 정보가 부족할 때 Agent 질문과 사용자 답변 입력을 제공한다.
- 등록 확인 상태: Agent가 추출한 최종 일정 정보를 요약하고 승인 여부를 묻는다.
- 완료 상태: 일정 및 알림 등록 결과를 보여준다.
- 실패 상태: 날짜 해석 실패, 잘못된 시간, 저장 실패 등 등록 불가 사유를 알려준다.

## 7. 기능 요구사항

### 자연어 일정 입력

- 사용자는 말하듯 일정 문장을 입력할 수 있어야 한다.
- 입력 문장은 날짜, 시간, 일정명, 장소, 알림 조건을 포함할 수 있다.
- 입력 예시는 다음을 지원해야 한다.
  - "내일 오후 3시 병원 예약"
  - "금요일 자정까지 과제 제출"
  - "다음 주 월요일 오전 10시 회의"

### 일정 정보 추출

- Agent는 자연어 입력에서 다음 정보를 추출해야 한다.
  - 일정명
  - 날짜
  - 시간
  - 장소
  - 알림 시간
- 날짜와 시간은 캘린더 저장 가능한 구조화 데이터로 변환되어야 한다.
- 알림 시간이 없으면 기본 알림을 제안하거나 적용해야 한다.

### 추가 질문 처리

- 날짜 또는 시간이 없으면 Agent는 추가 질문을 해야 한다.
- 애매한 날짜 표현은 추측해서 등록하지 않고 사용자 확인을 받아야 한다.
- 사용자의 추가 답변은 기존 입력 맥락과 합쳐 최종 일정 정보로 반영되어야 한다.

### 등록 전 요약 및 승인

- Agent는 실제 저장 전에 최종 등록 내용을 사용자에게 보여줘야 한다.
- 사용자 승인이 없으면 캘린더 등록과 알림 예약을 실행하지 않는다.
- 사용자가 취소하면 등록을 중단하고 홈 화면 흐름으로 돌아갈 수 있어야 한다.

### 캘린더 등록

- 승인된 일정은 앱의 캘린더 데이터에 저장되어야 한다.
- 저장된 일정은 캘린더 화면에서 날짜 기준으로 확인 가능해야 한다.
- MVP에서는 Google Calendar API 또는 로컬 저장소 중 구현 가능한 방식을 사용한다.

### 알림 예약

- 일정에 알림 시간이 있으면 알림을 예약해야 한다.
- 알림 시간이 없으면 기본 알림은 30분 전으로 처리한다.
- MVP 알림은 로컬 알림을 기준으로 한다.

## 8. Agent 동작 원칙

Agent는 사용자의 일정을 대신 정리하고 실행을 돕는 간결한 생활 비서처럼 동작해야 한다.

- 불필요하게 긴 설명을 하지 않는다.
- 등록 전 핵심 정보를 명확하게 확인한다.
- 애매한 정보가 있으면 추측하지 않고 질문한다.
- 사용자가 급하게 입력해도 자연스럽게 정리해준다.
- 사용자 승인 없는 자동 등록은 하지 않는다.
- 일정 수정, 삭제처럼 기존 데이터를 변경하는 행동은 MVP 범위에서 제외한다.

## 9. 예외 처리

| 상황                  | 처리 방식                     |
| --------------------- | ----------------------------- |
| 날짜 정보 없음        | 추가 질문                     |
| 시간 정보 없음        | 추가 질문                     |
| 애매한 날짜 표현      | 사용자 확인                   |
| 알림 미입력           | 기본 알림 30분 전 적용        |
| 잘못된 날짜 또는 시간 | 재입력 요청                   |
| 저장 실패             | 실패 사유 안내 후 재시도 안내 |
| 사용자 취소           | 등록 중단                     |

## 10. MVP 제외 범위

다음 기능은 MVP에서 제외한다.

- 음성 입력
- 복잡한 반복 일정
- 기존 일정 수정 및 삭제
- 여러 캘린더 계정 동기화
- 일정 공유
- 사용자의 하루 일정 자동 최적화
- 장기 메모리 기반 개인화 추천
- 사용자 승인 없는 자동 등록

## 11. 데모 성공 기준

아래 입력에 대해 Agentic Workflow가 정상 작동하면 MVP 데모 성공으로 본다.

- "내일 오후 3시 병원 예약"
- "금요일 자정까지 과제 제출"
- "다음 주 월요일 오전 10시 회의"

각 입력은 다음 흐름을 통과해야 한다.

1. 자연어 입력
2. 일정 정보 추출
3. 부족한 정보가 있으면 추가 질문
4. 등록 전 요약
5. 사용자 승인
6. 캘린더 저장
7. 알림 예약
8. 완료 메시지 표시
