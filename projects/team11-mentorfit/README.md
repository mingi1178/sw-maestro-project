# Mentor-Fit

팀에 적합한 멘토를 추천하는 AI 에이전트 시스템.

AI/SW 마에스트로 연수생 팀의 기술 스택, 목표, 멘토링 주제를 입력하면, Notion에 등록된 멘토 데이터를 분석해 적합한 멘토 후보와 추천 근거, 부족한 멘토링 영역, 함께 매칭하면 좋은 멘토 조합을 리포트로 제공한다.

## 배경

멘토 매칭 과정은 팀 소개 작성 ~1시간, 멘토 소개글 읽기 및 요약 ~5시간, 팀에 맞는 조합 찾기 ~4시간으로 **총 10시간 이상** 소요된다. Mentor-Fit은 이 과정을 2시간 이내로 단축하는 것을 목표로 한다.

## 핵심 기능

- **팀 프로필 생성**: 팀원 3명의 태그, 프로젝트 목표, 멘토링 희망 주제를 하나의 프로필로 통합
- **멘토 후보 검색**: 이미 매칭 완료된 멘토 제외 후, 태그 유사도 + 의미 기반 유사도로 1차 후보 추출
- **멘토 조합 생성**: 후보 멘토별 약점을 상호 보완하는 최적 멘토 조합 구성
- **추천 리포트 생성**: 추천 이유, 도움받을 수 있는 부분, 부족한 영역, 멘토 조합 제안

## 기술 스택

| 역할 | 기술 |
|------|------|
| Backend API | Python, FastAPI |
| LLM / Embedding | Upstage Solar API |
| Vector DB | Qdrant 또는 ChromaDB |
| UI | Streamlit |
| Pipeline | LangGraph 또는 순차 파이프라인 |
| 로컬 저장소 | SQLite 또는 JSON |
| 데이터 소스 | Notion API / CSV·Markdown 내보내기 |

## 시스템 파이프라인

```
팀 정보 입력
    ↓
팀 프로필 생성 (태그 통합·정규화)
    ↓
멘토 후보 검색 (태그 유사도 + Solar 임베딩 기반 의미 유사도)
    ↓
후보 멘토 별 조합 생성 (LLM이 멘토 소개글 × 팀 프로필 비교)
    ↓
추천 리포트 생성
```

## 시작하기

```bash
# 의존성 설치 (uv가 가상환경을 자동으로 관리)
uv sync

# 환경변수 설정 (.env.example 참고)
cp .env.example .env
# .env 파일에 UPSTAGE_API_KEY 입력

# FastAPI 개발 서버 실행
uv run uvicorn app.main:app --reload

# Streamlit UI 실행 (향후)
uv run streamlit run ui/search_ui.py
```

## 제약 사항

- AI 추천은 참고 자료이며, 최종 멘토 확정과 연락은 사용자가 직접 수행
- 멘토 소개글이나 태그가 부족한 경우 근거 없이 추정하지 않고 "추가 확인 필요"로 표시
- 이미 매칭 완료된 멘토 목록이 최신 상태가 아닐 수 있으며, 이 경우 리포트에 경고 표시
