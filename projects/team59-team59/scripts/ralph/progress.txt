## Codebase Patterns
- Frontend: Next.js 14 with App Router in frontend/src/app/
- Backend: FastAPI in backend/main.py, Python 3.12
- Docker: hwpx-mcp uses node:20-slim official image (no custom Dockerfile needed)
- TypeScript typecheck command: `npm run typecheck` (alias for `tsc --noEmit`)
- Python typecheck: `uv run --with mypy --with fastapi --with httpx mypy main.py mcp_client.py --ignore-missing-imports` (system uses uv, no venv)
- Session state stored in module-level `_session_last_used: dict[str, float]` in backend/main.py
- Session dirs: /tmp/sessions/{session_id}/ — created via `touch_session()`, cleaned via `cleanup_expired_sessions()`
- MCP client in backend/mcp_client.py: HTTP JSON-RPC primary, stdio subprocess fallback
- FastAPI lifespan used for MCPClient init/teardown; `_mcp_client: MCPClient | None` is module-level global
- MCP JSON-RPC paths tried in order: /, /rpc, /mcp (hwpx-mcp endpoint unknown ahead of time)
- Next.js 14 does NOT support `next.config.ts`; must use `next.config.mjs` or `next.config.js`
- Frontend session ID: stored in localStorage under key `hwp-session-id`, injected as `X-Session-ID` header
- NEXT_PUBLIC_BACKEND_URL env var used for backend URL (default: http://localhost:8000)
- File upload: POST /upload with FormData (no explicit Content-Type header — browser sets multipart boundary)
- Message role 'info' = green success bubble; 'system' = red error bubble
- Claude agentic loop: _MAX_TOOL_ROUNDS=10; check stop_reason=="tool_use" to continue; build serialisable dicts for messages (not SDK objects)
- MCP inputSchema → Claude input_schema: always remap camelCase when converting tools for Claude API
- Use getattr(block, "type", None) for type-safe access to Anthropic content blocks (avoids mypy errors with union types)

## 2026-05-08 - US-007
- What was implemented: POST /chat endpoint — core Claude API ↔ hwpx-mcp agentic loop
  - `ChatRequest` Pydantic model: `{ message: str, session_id: str }`
  - `_claude_client: anthropic.AsyncAnthropic` initialized in lifespan, closed on shutdown
  - `/chat` endpoint: fetches MCP tools, converts inputSchema→input_schema, builds system prompt with session file path
  - Agentic loop (up to 10 rounds): calls Claude, detects tool_use stop_reason, executes tools via `_mcp_client.call_tool`, feeds results back as tool_result messages
  - Returns `{ response: str, file_ready: bool }` — `file_ready` is true when `current.hwpx` exists in session dir
  - Degrades gracefully: empty tools list if MCP unavailable; tool errors returned as string content blocks
- Files changed:
  - backend/main.py (import json/anthropic, _claude_client global, lifespan update, ChatRequest, /chat endpoint)
- **Learnings for future iterations:**
  - Serialize response.content blocks manually to dicts (not SDK objects) before appending to messages list
  - MCP tools use camelCase `inputSchema`; Claude API requires snake_case `input_schema` — always remap
  - Use `getattr(block, "type", None)` to access content block type without mypy union errors
  - AsyncAnthropic client has `.close()` coroutine — must be awaited in lifespan teardown
  - ANTHROPIC_API_KEY read automatically from env by AsyncAnthropic(); no explicit kwarg needed
