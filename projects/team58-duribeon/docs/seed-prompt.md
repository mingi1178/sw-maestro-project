# 두리번 시드 데이터 생성 작업

역할: 너는 서울 전역의 골목·로컬 문화에 정통한 큐레이터다. 가이드북에 안 나오는 "이 동네에서만 가능한" 작은 장소들을 체계적으로 발굴해 데이터베이스로 만드는 일을 한다.

작업: 서울의 매력적인 동네 **12곳**을 선정하고, 각 동네마다 가이드북·인스타 톱10에 잘 등장하지 않는 실재 장소 **8곳씩**을 발굴해 단일 JSON 객체로 출력한다. 총 장소 수 **96곳 목표** (최소 60곳 이상).

## 수행 방법 — 반드시 따를 것

1. **웹 검색을 적극 활용**해서 각 후보 장소의 다음을 검증한다:
   - 현재 영업 중인지 (폐업·이전·휴업이면 제외)
   - 정확히 해당 동네 안에 있는지 (행정동/법정동 둘 다 OK)
   - 가이드북·블로그 톱10에 이미 진입했는지 (진입했다면 제외)
2. 검증되지 않거나 정보가 부족한 곳은 **절대 환각으로 채우지 마라**. 빼라.
3. 한 동네에서 8곳을 못 찾으면 6~7곳만 채우고, 빈 슬롯은 만들지 마라.

## 동네 선정 — 서울 전역에서 다양성 있게

다음 카테고리를 골고루 포함하되, 가이드북 메인 코스(명동·경복궁·N서울타워·강남역·롯데월드 등)는 동네 선정 자체에서 제외:

- **한옥·노포 골목** (예: 익선동, 서순라길, 통의동)
- **공장 리모델링·인더스트리얼** (예: 성수동, 문래동)
- **외국인+로컬 혼합** (예: 연남동, 한남동, 이태원 뒷골목)
- **시장 중심** (예: 망원시장 주변, 광장시장 뒷골목, 통인시장)
- **산책 골목** (예: 부암동, 성북동, 삼청동 뒷골목, 서촌)
- **새롭게 뜨는 동네** (예: 을지로 3가, 약수, 후암동, 해방촌)
- **대학가 안골목** (예: 신촌·홍대 뒷골목, 혜화 뒷골목)

각 동네 안에서 카테고리 다양화 — `food` / `place` / `experience` 모두 최소 1개 이상.

## 장소 단위 선정 기준

- 가이드북·블로그 톱10에 안 나오는 골목 단위 장소
- 작은 공방, 노포, 50년 이상 된 가게, 작가 운영 공간 등 디테일 살아있는 곳
- 그 동네에서만 가능한 우연성·로컬리티
- 안전 (사유지 침입 X, 종교시설 안 게임적 행위 X, 야간 단독 외진 곳 X)

## 출력 형식

단 **하나의 JSON 객체**만 출력. 마크다운 코드펜스(\`\`\`) 금지. 인사·해설·설명 일절 금지.

```jsonc
{
  "version": 1,
  "areas": [
    {
      "id": "snake_case 영문 (예: ikseon, mangwon, mullae)",
      "name_ko": "한국어 동네명 (예: 익선동, 망원동, 문래동)",
      "name_en": "영어 동네명 (예: Ikseon-dong, Mangwon-dong)",
      "match_ko": ["사용자가 입력했을 때 매칭할 한국어 키워드 1~3개"],
      "match_en": ["1~3개 영어 소문자 키워드"]
    }
  ],
  "places": [
    {
      "id": "{area_id}_NN (두 자리 일련번호, 예: ikseon_01)",
      "area": "{areas[].id 중 하나와 정확히 일치}",
      "name_ko": "한국어 가게/장소 이름",
      "name_en": "영어 또는 자연스러운 로마자 표기",
      "category": "food | place | experience",
      "tags": ["회피 필터에 매칭되는 단어 배열"],
      "desc_ko": "30자 내외 한 줄 설명",
      "desc_en": "Same idea in ~30-50 chars natural English",
      "offbeat_score": 0.55~0.95
    }
  ]
}
```

## 규칙 — 빠짐없이 채울 것

- areas 와 places의 모든 필드 필수. `null` / 빈 배열 / 빈 문자열 금지.
- `places[].area`는 반드시 `areas[].id` 중 하나와 정확히 일치.
- 각 area마다 places 최소 6곳, 최대 10곳.
- id 형식: areas는 snake_case 영문, places는 `{area_id}_01`부터 두 자리 일련번호.
- 같은 id 절대 중복 X (areas, places 각각 unique).
- `desc_ko`는 30자 내외, `desc_en`은 30~50자 내외. 두 desc가 같은 사실을 자연스러운 톤으로 (직역 X).
- `name_en`은 실제 영문 간판이 있으면 그대로 사용, 없으면 자연스러운 로마자.
- `tags`는 다음 어휘 풀에서만 골라 (회피 필터와 호환되는 단어):
  - **음식**: `food`, `snack`, `bakery`, `noodles`, `dumplings`, `seafood`, `spicy`, `no-spicy`, `warm`
  - **음료/술**: `cafe`, `bar`, `wine`, `drink`, `drink-light`
  - **분위기/카테고리**: `old`, `hanok`, `retro`, `vinyl`, `music`, `history`, `industrial`, `quiet`, `couple`, `craft`, `photo`, `bookshop`, `japanese`, `calligraphy`, `stationery`, `pottery`, `shoes`, `rooftop`, `local`, `market`, `alley`, `factory`, `gallery`, `artisan`, `traditional`, `vintage`, `family`, `solo`
- `offbeat_score` 가이드:
  - `0.55~0.70` — 알 만한 사람은 아는 곳 (블로그 일부 노출)
  - `0.70~0.85` — 동네 단위 로컬 추천 (검색해야 나오는 곳)
  - `0.85~0.95` — 골목 끝, 간판도 작은 공간 (현지인 입소문)

## 출력 직전 자가 검증 — 모두 통과해야 출력

1. 각 `place.area`가 `areas[]`의 id 중 하나와 정확히 일치하는가?
2. 각 area마다 places가 최소 6곳 이상인가?
3. 모든 필드가 채워져 있는가? (null / 빈 배열 / 빈 문자열 금지)
4. 같은 area 안에 `food` / `place` / `experience` 카테고리가 모두 최소 1개 있는가?
5. id 충돌이 없는가? (areas끼리, places끼리 각각 unique)
6. 폐업·이전·환각 장소가 섞여 있지 않은가? (검색 결과로 검증된 곳만)
7. tags가 위 어휘 풀 안에서만 사용됐는가?
8. `offbeat_score`가 0.55~0.95 범위 안에 있는가?

이 8개 검증을 통과한 후에만 JSON을 출력하라. 통과 못한 항목이 있으면 해당 부분을 수정한 뒤 다시 검증하라. 통과 못한 채로 출력 금지.

JSON 외 텍스트(인사, 설명, 해설, 마크다운 펜스)는 일절 출력 금지.

지금 시작. 검색을 켜고, 동네 후보부터 발굴해라.
