# <a name="_66rskn8gi764"></a>**[13조] AI 기반 개인 지식 그래프 에이전트 프로젝트 기획서**

참여자 : 김민재 김민형 남동완 이수민 이태석
## <a name="_pbhb5lpiokdu"></a>**1. 문제 정의 및 프로젝트 개요**

|<p>- **프로젝트 한 줄 정의(**프로젝트 전체를 한 문장으로 요약) [필수]</p><p>&emsp;- 여러 생활 도메인의 개인 데이터를 단일 지식 그래프로 통합하고, 복수의 도메인 전문 AI 에이전트가 자율적으로 교차 패턴을 분석·종합하여 실행 가능한 인사이트를 제공하는 개인용 AI 지식 관리 시스템</p><p>- **서비스 한 줄 정의**(사용자 관점에서 이 서비스가 무엇인지 한 문장으로 설명) [필수]</p><p>&emsp;- 내 건강, 학업, 금융, 취미 데이터를 자동으로 수집·연결하고, AI 에이전트들이 서로 토론하며 내가 스스로 발견하기 어려운 삶의 패턴과 맞춤형 행동 계획을 제안해 주는 개인 지식 관리 서비스</p><p>- **서비스 선정 배경(**왜 이 주제를 선택했는가) [권장]</p><p>&emsp;- 현대인의 생활 데이터는 건강, 일정, 재무, 창작 등 수많은 앱에 분산되어 있으나 도메인 간 연결이 없어 사용자가 교차 패턴을 직접 발견하기 어렵다. 각 앱에서 개별 인사이트는 얻더라도, "수면 부족이 업무 생산성에 미치는 영향" 또는 "스트레스와 소비 패턴의 연관성"과 같은 복합적 패턴은 사용자가 수동으로 연결해야만 한다. AI 에이전트 기술로 이 과정을 자동화하면 사용자에게 전혀 새로운 자기 이해 경험을 제공할 수 있다는 가설에서 출발했다.</p><p>- **해결하려는 문제**(핵심으로 생각하는 문제, 문제라고 생각한 이유를 함께 기재) [필수]</p><p>&emsp;- ① 분산된 개인 데이터 통합 문제<br>&emsp;  `   `건강, 일정, 금융, 사진 등 서로 다른 시스템에 흩어진 개인 데이터를 하나의 공간에서 의미 있게 연결하기 어렵다.<br>&emsp;  ② 교차 도메인 패턴 발견의 어려움<br>&emsp;  `   `"수면 부족 → 업무 지연", "스트레스 증가 → 소비 증가"와 같은 도메인 간 인과관계를 사용자가 직접 발견하는 데 높은 인지 부하가 필요하다.<br>&emsp;  ③ 지식 관리의 수동성<br>&emsp;  `   `데이터 기록·태깅·연결 등 지식 관리 작업 대부분이 여전히 수동으로 이루어진다.</p><p>- **대상 사용자:** (누가 이 Agent를 사용하는가?) [필수]</p><p>&emsp;- 자기 관리에 관심이 있고, 여러 생활 도메인(건강·학업·재무·취미)의 데이터를 하나의 공간에서 통합해 인사이트를 얻고자 하는 성인 사용자. 스마트 기기 생태계를 적극적으로 활용하고 있으며, 개별 앱에서는 각자의 인사이트를 얻고 있으나 도메인을 아우르는 통합적 시각이 없어 불편함을 느끼는 이들.</p><p>- **핵심 가치:** (이 프로젝트가 사용자에게 주는 가장 큰 이점) [필수]</p><p>&emsp;- 사용자가 직접 연결하지 않아도 AI가 생활 데이터의 교차 패턴과 인과관계를 자동으로 발견하고, 복수의 도메인 에이전트가 서로 논쟁·검증한 결과를 바탕으로 실행 가능한 인사이트를 제공하는 것.</p>|

## <a name="_v1z5xj52uoz3"></a>**2. 사용자 및 Agent 설계**

|<p>- **타깃 사용자 페르소나(ex, 연령대, 직업, 특징 등)** [권장]</p><p>&emsp;- 20~40대 / 자기계발에 관심 있는 직장인 또는 학생<br>&emsp;  스마트 기기 활용도가 높으며 건강·일정·재무 앱을 이미 사용 중이지만, 앱 간 데이터 단절로 인해 전체적인 생활 패턴을 파악하지 못하는 사람.<br>&emsp;  수동 데이터 입력보다는 자동 연동을 선호하며, AI가 먼저 패턴을 찾아 제안해 주는 경험에 관심 있음.</p><p>- **Agent의 역할(최대한 구체적으로 기재)** [필수]</p><p>&emsp;- 1. 입력 처리 에이전트<br>&emsp;  `   `사용자의 자유 텍스트·파일(PDF 등) 입력을 분석해 핵심 개념을 추출하고 분류·태깅된 지식 노드로 자동 변환한다.<br>&emsp;  2. 도메인 전문가 에이전트 (페르소나)<br>&emsp;  `   `건강·학업·금융·취미 각 도메인을 담당하는 전문 에이전트가 자신의 영역 데이터를 분석하고, 사용자 질의를 관련 페르소나에 자동 라우팅한다.<br>&emsp;  3. 노드 관리 에이전트<br>&emsp;  `   `신규 노드를 분석해 의미적으로 유사한 노드를 자동 클러스터링하고 도메인 간 관계 엣지를 자동 생성한다.<br>&emsp;  4. 토론 오케스트레이터 에이전트<br>&emsp;  `   `교차 도메인 패턴 감지 시 복수의 도메인 에이전트를 병렬 호출해 3라운드 구조화 토론(분석 → 반론 → 합성)을 진행하고 합성 결과 및 실행 계획 노드를 생성한다.</p><p>- **Agent의 성격 및 톤앤매너** [권장]</p><p>&emsp;- 분석적이되 친근하고 이해하기 쉬운 언어를 사용한다. 각 도메인 에이전트는 자신의 영역에서 자신감 있게 분석하되, 토론 과정에서 다른 도메인 에이전트의 시각을 존중하며 합리적으로 반론한다. 최종 결과는 원인 분석과 구체적 행동 항목을 명확하게 제시한다.</p><p>- **Agent의 자율성 범위:** (어디까지 스스로 결정하고 실행할 것인가? 예: 정보 검색 후 요약까지만 수행) [필수]</p><p>&emsp;- 자율 실행 영역:<br>&emsp;  `  `- 노드 클러스터링 및 도메인 간 자동 연결 생성<br>&emsp;  `  `- 교차 도메인 패턴 감지 및 토론 자동 트리거 (일일 1회 주기 제한)<br>&emsp;  `  `- 실행 계획 노드 및 시스템 리마인더 자동 생성 (최우선 항목 1건)<br>&emsp;  사용자 승인 필요:<br>&emsp;  `  `- 새로운 도메인 페르소나 생성 (시스템이 제안, 사용자가 최종 결정)<br>&emsp;  인간 개입(HITL):<br>&emsp;  `  `- 에이전트 간 합의 미달 시 사용자에게 명확화 질문을 요청해 추가 입력을 받은 후 최종 결론 도출</p>|

## <a name="_ur9kajfbkxt5"></a>**3. 핵심 기능 및 사용자 흐름**

|<p>- **주요 사용자 시나리오** [필수]</p><p>&emsp;- 시나리오 1 — "요즘 왜 이렇게 피곤하지?"<br>&emsp;  사용자가 피로감을 텍스트로 입력한다. AI가 이를 건강 노드로 저장하고 같은 시기의 수면 데이터·캘린더 일정 노드와 자동 연결한다. 교차 패턴(수면×업무)이 감지되면 건강 에이전트와 업무 에이전트가 토론을 시작해 "회의 증가 → 수면 감소 → 피로 누적"의 인과관계를 도출하고 구체적 행동 계획 노드를 생성한다.<br>&emsp;<br>&emsp;  시나리오 2 — "이번 달 지출이 왜 이렇게 많지?"<br>&emsp;  사용자가 금융 데이터를 동기화하면 금융 에이전트가 지출 패턴을 분석한다. 같은 시기의 스트레스 관련 건강 지표와 교차 분석해 "스트레스 급증 기간의 충동 소비 증가" 패턴을 발견하고 개선 행동 항목을 제안한다.</p><p>- **핵심 기능 정의** [필수]</p><p>&emsp;- 1. 스마트 입력 및 자동 노드 생성<br>&emsp;  `   `자유 텍스트·파일 입력 → AI 핵심 개념 추출 → 분류·태깅된 노드 자동 생성<br>&emsp;  2. 외부 데이터 자동 동기화<br>&emsp;  `   `OAuth 기반 외부 서비스(Google Calendar, Apple Health 등)와 연동하여 데이터를 주기적으로 수집하고 노드로 변환<br>&emsp;  3. 인터랙티브 지식 그래프 캔버스<br>&emsp;  `   `노드와 연결선을 시각적으로 탐색하는 그래프 뷰 (드래그·줌·선택·필터 지원)<br>&emsp;  4. 도메인별 AI 페르소나 채팅<br>&emsp;  `   `자연어 질문 → 관련 페르소나 자동 라우팅 → 도메인 특화 응답; 복수 페르소나 동시 대화 가능<br>&emsp;  5. 멀티 에이전트 구조화 토론<br>&emsp;  `   `교차 도메인 패턴 감지 시 자동/수동 토론 시작 → 3라운드 진행 → 합성 결과 및 실행 계획 생성<br>&emsp;  6. 자동 클러스터링·연결<br>&emsp;  `   `AI가 의미적 유사성 기반으로 노드를 클러스터링하고 도메인 간 관계 엣지 자동 생성<br>&emsp;  7. 주간 인사이트 리포트<br>&emsp;  `   `매주 지난 7일간의 활동 패턴을 요약한 인사이트 노드를 자동 생성</p><p>- **워크플로우**</p><p>&emsp;- **사용자 관점 워크플로우** [필수]</p><p>&emsp;&emsp;- ① 웹 서비스 접속 및 도메인 페르소나 설정<br>&emsp;&emsp;  ② 외부 데이터 소스(캘린더·건강·사진 등) 동기화 권한 허용<br>&emsp;&emsp;  ③ 텍스트·파일 입력 또는 자동 동기화로 노드 생성<br>&emsp;&emsp;  ④ 그래프 캔버스에서 노드·연결선 시각적 탐색<br>&emsp;&emsp;  ⑤ 특정 노드 선택 후 페르소나에게 자연어 질문<br>&emsp;&emsp;  ⑥ 교차 도메인 패턴 감지 시 멀티 에이전트 토론 시작 및 결과 확인<br>&emsp;&emsp;  ⑦ 토론 결과로 생성된 실행 계획 확인 및 리마인더 등록</p><p>&emsp;- **시스템 관점 워크플로우** [권장]</p><p>&emsp;&emsp;- 사용자 입력/동기화 이벤트 발생<br>&emsp;&emsp;  → 입력 처리 에이전트: 텍스트 분석 및 노드 생성<br>&emsp;&emsp;  → 노드 관리 에이전트: 클러스터링·도메인 간 자동 연결 수행<br>&emsp;&emsp;  → 페르소나 라우터: 쿼리를 관련 도메인 에이전트에 배분<br>&emsp;&emsp;  → (패턴 감지 시) 토론 오케스트레이터 작동<br>&emsp;&emsp;  `   `Round 1: 도메인별 병렬 독립 분석<br>&emsp;&emsp;  `   `Round 2: 교차 도메인 반론<br>&emsp;&emsp;  `   `Round 3: 서버 측 합성 (Solar LLM API 기반 교차 도메인 결과 통합)<br>&emsp;&emsp;  → (합의 미달 시) 사용자 역질문(HITL) 후 재합성<br>&emsp;&emsp;  → 최종 결과 노드·엣지·리마인더 생성</p>|

## <a name="_cmo9spdunyv4"></a>**4. 기술 구현 설계**

|<p>- **기술 스택** [필수]</p><p>&emsp;- - 플랫폼: 웹 서비스<br>&emsp;      - UI 프레임워크: 웹 프레임워크 기반 반응형 UI (React + TypeScript)<br>&emsp;      - 데이터 저장: 서버 기반 영구 저장 (관계형 DB + 그래프 DB)<br>&emsp;      - AI 추론: Solar LLM API (Upstage Solar) 기반 멀티 에이전트 추론<br>&emsp;      - 외부 데이터 연동: OAuth 기반 외부 서비스 연동 (Google Calendar, Apple Health API 등), REST API 기반 데이터 수집<br>&emsp;      - 비동기 처리: 서버 비동기 처리 기반 멀티 에이전트 병렬 실행 (Spring Async / WebFlux)</p><p>- **시스템 아키텍처** [권장]</p><p>&emsp;- Frontend (React: 그래프 캔버스 · 채팅 UI · 입력 모달)<br>&emsp;  `   `↕  양방향 바인딩<br>&emsp;  REST API / WebSocket (Spring Boot: 상태 관리 · 이벤트 처리)<br>&emsp;  `   `↕<br>&emsp;  Service Layer (에이전트 서비스 집합)<br>&emsp;  `  `· 노드 관리 에이전트  · 토론 오케스트레이터  · 페르소나 라우터  · 동기화 서비스<br>&emsp;  `   `↕                          ↕<br>&emsp;  Data Layer (PostgreSQL + 그래프 DB)      AI Layer (Solar LLM API)<br>&emsp;<br>&emsp;  ※ 교차 도메인 합성은 서버 AI Layer에서 수행; 민감 데이터는 암호화 전송 및 서버 내 격리 처리</p><p>- **프롬프트 설계 전략** [권장]</p><p>&emsp;- - 역할 분리: 각 도메인 에이전트(건강·학업·금융·취미·업무)는 별도 시스템 프롬프트를 가지며 자신의 도메인 관점에서만 분석하도록 제한<br>&emsp;      - 구조화 출력: 클러스터 생성·토론 합성 등 파싱이 필요한 단계는 JSON 스키마 가이드 또는 구조체 타입 추론을 활용해 신뢰성 확보<br>&emsp;      - 반론 강제: Round 2에서 상대 에이전트 분석에 의도적으로 반론하도록 설계해 동조적 응답(sycophancy) 방지<br>&emsp;      - 프라이버시 보호: Solar API 전송 시 개인 식별 정보를 제거한 익명화 데이터만 전달; 서버 측 암호화 저장으로 데이터 보호</p><p>- **데이터 활용 및 기억 관리** [권장]</p><p>&emsp;- - 영구 저장: 모든 노드·엣지·채팅 기록·토론 기록을 서버 DB에 영구 저장<br>&emsp;      - 페르소나 범위 분리: 각 노드는 하나 이상의 페르소나 ID와 연결되어 도메인별 컨텍스트 분리 유지<br>&emsp;      - 처리 상태 추적: 노드별 처리 완료 플래그로 AI 분석 완료 여부를 관리하고 미처리 노드만 에이전트에 전달해 중복 분석 방지<br>&emsp;      - 외부 ID 매핑: 캘린더·건강·사진 등 외부 시스템의 고유 ID를 저장해 중복 동기화 방지</p><p>- **제약사항 및 예외 처리** [필수]</p><p>&emsp;- - Solar LLM API 장애 환경: Solar API 호출 실패 시 재시도 로직(Retry) 및 지수 백오프 적용; 지속 실패 시 사용자에게 명확한 오류 메시지 제공<br>&emsp;      - 외부 데이터 권한 미허용: 권한이 없는 데이터 소스는 건너뛰고 허용된 소스만으로 동작<br>&emsp;      - 토론 실패: 개별 에이전트 또는 합성 단계 실패 시 이미 생성된 부분 결과 기반 폴백 합성 결과 제공<br>&emsp;      - 이번 구현 제외 범위: 모바일 네이티브 앱 지원, 외부 사용자와의 데이터 공유 기능, 유료 결제 시스템</p>|

## <a name="_nzdjprj1rbb0"></a>**5. 성과 평가 및 실행 계획**

|<p>- **성공 지표(KPI)** [권장]</p><p>&emsp;- - 전체 agentic workflow 실행 가능 여부: 스마트 입력 → 노드 생성 → 그래프 시각화 → 페르소나 채팅 → 멀티 에이전트 토론 → 결과 노드 생성의 흐름이 오류 없이 동작<br>&emsp;      - 교차 도메인 패턴 감지 및 자동 토론 트리거: 서로 다른 도메인의 노드가 충분히 쌓였을 때 토론이 정상 시작됨<br>&emsp;      - Solar API 기반 실행 완결성: Solar LLM API 연동을 통해 전체 워크플로우가 안정적으로 동작</p><p>- **MVP 범위** [필수]</p><p>&emsp;- MVP 범위:<br>&emsp;  `  `· 스마트 텍스트 입력 및 노드 자동 생성<br>&emsp;  `  `· 최소 2개 외부 데이터 소스 동기화 (캘린더 + 건강 데이터)<br>&emsp;  `  `· 인터랙티브 지식 그래프 캔버스<br>&emsp;  `  `· 최소 2개 도메인 페르소나 (건강·학업)와의 채팅<br>&emsp;  `  `· 멀티 에이전트 토론 (자동 또는 수동 트리거)<br>&emsp;  `  `· 웹 서비스 환경에서 Solar LLM API 기반 전체 agentic workflow 실행<br>&emsp;<br>&emsp;  추가 기능:<br>&emsp;  `  `· 클라우드 동기화 및 다중 기기 지원<br>&emsp;  `  `· 금융 데이터 자동 파싱 (수동 입력은 허용)<br>&emsp;  `  `· 사진 AI 분석 기능<br>&emsp;  `  `· 주간 인사이트 자동 생성 (시간 여유 시 추가)</p><p>&emsp;</p><p>- **단계별 개발 로드맵** [필수]</p><p>&emsp;- 1단계: 기반 구축<br>&emsp;  `  `데이터 모델 설계 (노드·엣지·페르소나), 그래프 캔버스 UI, 텍스트 입력 및 수동 노드 생성<br>&emsp;  2단계: AI 연동<br>&emsp;  `  `Solar LLM API 연동, 입력 텍스트 분석 및 자동 노드 생성, 기본 페르소나 채팅<br>&emsp;  3단계: 에이전트화<br>&emsp;  `  `노드 관리 에이전트 (클러스터링·자동 연결), 외부 데이터 동기화 (캘린더·건강 데이터)<br>&emsp;  4단계: 멀티 에이전트 토론<br>&emsp;  `  `토론 오케스트레이터 구현, 3라운드 토론 구조, HITL(인간 개입) 흐름<br>&emsp;  5단계: 완성도 향상<br>&emsp;  `  `폴백 처리, 예외 처리 강화, 데모 시나리오 검증 및 최종 점검</p><p>- **기대 효과** [권장]</p><p>&emsp;- 사용자는 여러 앱에 흩어져 있던 생활 데이터를 하나의 공간에서 통합해 볼 수 있으며, AI 에이전트들이 자동으로 교차 도메인 패턴을 발견해 사용자가 스스로 발견하기 어려운 인사이트를 제공한다. 멀티 에이전트 토론 구조를 통해 단일 AI 답변보다 다각적이고 검증된 분석 결과를 경험할 수 있으며, 실행 계획 노드를 통해 인사이트가 바로 행동으로 이어지는 사용자 경험을 구현한다.</p>|
