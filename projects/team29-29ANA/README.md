# 🎙️ AI 면접 시뮬레이터

이력서/자소서 PDF와 채용공고(JD)를 입력하면, AI가 실제 면접관처럼 질문을 생성하고 답변을 평가하여 종합 리포트를 제공하는 에이전틱 RAG 기반 면접 연습 서비스입니다.

## 주요 특징

- **9개 직군 자동 분류** — JD를 분석해 engineering, design, product, marketing 등 직군을 자동 판별하고, 직군별 면접관 페르소나·질문 톤·카테고리 라벨을 분기합니다.
- **RAG 기반 질문 생성** — 이력서를 벡터 DB에 임베딩하고, 질문마다 관련 청크만 검색해 LLM에 주입합니다. 이력서의 특정 프로젝트·수치·역할을 자연스럽게 인용하는 질문이 생성됩니다.
- **4축 답변 평가 + 교차검증** — STAR 구조, 구체성, 직무 관련성, 일관성을 Rubric + Chain-of-Thought + Few-shot으로 평가합니다. 답변에서 주장한 경험을 자소서 RAG로 교차검증해 역할 부풀림 같은 불일치를 감지합니다.
- **적응형 꼬리질문** — 약점이 발견되면 최대 3단계까지 꼬리질문으로 깊이 파고들고, 약점이 없으면 다음 카테고리로 자연스럽게 넘어갑니다.
- **난이도 자동 캘리브레이션** — 최근 답변 평가 점수를 기반으로 다음 질문 난이도를 1~5 사이에서 자동 조정합니다. 잘 답하면 더 어렵게, 못 답하면 더 쉽게.

## 아키텍처

```
이력서 PDF ──→ document_loader ──→ ChromaDB (벡터 임베딩)
                                        ↑ RAG 검색
채용공고 JD ──→ job_classifier ──→ job_family (9개 직군)
회사명 ──────→ tavily_search ──→ CompanyProfile (기술스택·문화·뉴스)
                                        │
              ┌─────────────────────────┘
              ▼
        SessionContext (컨텍스트 통합)
              │
              ▼
    ┌─── orchestrator (에이전트 루프) ───┐
    │                                    │
    │  question_generator                │
    │    ↕ (질문 → 답변 → 평가 반복)       │
    │  answer_evaluator                  │
    │    │                               │
    │    ├─ 약점 있음 → 꼬리질문           │
    │    ├─ 약점 없음 → 다음 메인 질문      │
    │    └─ 충분히 진행 → 종료 제안         │
    │                                    │
    └────────────────────────────────────┘
              │
              ▼
        report_generator (종합 리포트)
```

## 에이전틱 워크플로우

이 시스템은 단순한 1회성 LLM 호출이 아니라, AI가 상태를 유지하면서 **판단 → 행동 → 관찰 → 다시 판단**을 반복하는 에이전틱 워크플로우입니다.

| 에이전트 요소 | 구현 |
| --- | --- |
| 자율적 판단 | 평가 결과의 weakness_tags를 보고 꼬리질문/메인질문/종료를 AI가 결정 |
| 상태 유지 | `SessionState.history`에 (질문, 평가결과) 튜플이 누적 |
| 도구 사용 | ChromaDB RAG, Tavily 웹 검색, Solar LLM |
| 루프 | 질문 → 답변 → 평가 → 판단 → 다음 질문 반복 |

## 기술 스택

| 영역 | 기술 | 용도 |
| --- | --- | --- |
| LLM | Upstage Solar (OpenAI 호환 API) | 질문 생성, 답변 평가, 직무 분류, 리포트 |
| 벡터 DB | ChromaDB | 이력서 임베딩 + 의미 검색 (RAG) |
| 웹 검색 | Tavily API | 지원 기업 정보 수집 |
| PDF 파싱 | pypdf | PDF → 텍스트 추출 + 청크 분할 |
| 언어 | Python 3.11+ | — |

## 평가 체계

### 4축 평가 기준

| 축 | 기준 | 5점 예시 | 1점 예시 |
| --- | --- | --- | --- |
| STAR 구조 | S/T/A/R 4요소 | 4요소 모두 구체적으로 서술 | 의견만 진술, 구체 사례 없음 |
| 구체성 | 수치·고유명사 | 수치 2개+ 고유명사 3개+ | 추상 표현만 |
| 직무 관련성 | JD 연결 | JD 핵심 요구와 직결 | JD와 무관 |
| 일관성 | 내부 + 자소서 | 모순 없고 자소서와 일치 | 자소서와 정면 모순 |

### 약점 태그

약점이 발견되면 해당 태그가 부여되고, 꼬리질문의 정조준 방향으로 사용됩니다:

> 구체성 부족 · 근거 부족 · 트레이드오프 미언급 · 결과 누락 · 역할 불명확 · 직무 관련성 약함 · 논리 비약 · 이전 답변과 모순 · 기술 깊이 부족 · 추상적 표현 과다 · 이력서와 답변 불일치
> 

### LLM 평가 신뢰도 강화 기법

| 기법 | 효과 |
| --- | --- |
| Rubric | 각 점수(1~5)를 행동 기준으로 정의. 느낌이 아닌 근거로 채점 |
| Chain-of-Thought | reasoning을 먼저 작성 후 점수 매김. 일관성 향상 |
| Few-shot | 강한/약한 답변 예시 2개를 프롬프트에 포함. 채점 패턴 고정 |
| 태그 화이트리스트 | 11개 표준 태그만 허용. 자유 텍스트 흔들림 방지 |
| 점수 클램프 | LLM이 6점, 0점 반환해도 1~5 강제. 타입 안전 |

## 직군별 페르소나

JD를 분석해 9개 직군 중 하나로 자동 분류하고, 각 직군에 맞는 면접관 페르소나가 적용됩니다:

| 직군 | 페르소나 | 질문 톤 예시 |
| --- | --- | --- |
| engineering | 엔지니어 | "p99 응답시간을 230ms까지 줄이셨다고 하셨는데, 어디가 병목이었어요?" |
| marketing | 마케팅 리드 | "ROAS가 가장 안 좋았던 캠페인 하나 들려주세요. 뭐가 문제였어요?" |
| design | 디자인 리드 | "데이터가 A안을 가리키는데 직감은 B안일 때 어떻게 결정하세요?" |
| product | 프로덕트 리드 | "KPI 못 친 기능 하나만 들려주세요. 회고는 어떻게 했어요?" |
| sales_bd | 세일즈 리드 | "클로징 못 한 딜 하나 떠올려주세요. 무엇이 결정적이었어요?" |
| operations | 운영 리드 | "가장 크게 개선한 프로세스 하나 들려주세요. 어디가 병목이었어요?" |
| hr_people | 피플 리드 | "구성원 갈등을 중재했던 경험이 있다면요?" |
| finance | 파이낸스 리드 | "결산 중에 큰 이슈를 발견했던 경험이 있다면요?" |
| general | 시니어 매니저 | "최근 1년 안에 가장 자랑스러웠던 성과 하나만 들려주세요." |

## 핵심 설계 결정

### RAG — 이력서 전체를 넣지 않는 이유

이력서 전체(수천~수만 토큰)를 매번 프롬프트에 넣으면 비용 폭증 + 관련 없는 내용이 노이즈가 됩니다. 대신 ChromaDB에 청크 단위로 임베딩하고, 질문/답변 키워드로 의미 검색해 관련 청크 3개만 주입합니다.

### 직무 분류 — 키워드 우선, LLM 폴백

대부분의 JD는 키워드 매칭만으로 직군 분류가 가능합니다. LLM을 매번 호출하면 비용과 지연이 발생하므로, 키워드로 명확하면 즉시 결정하고 모호할 때만 LLM에 위임합니다.
